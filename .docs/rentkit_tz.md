# Техническое задание (ТЗ)
# **RentKit — каталог недвижимости**

---

## **1. Общая информация о проекте**
**Название:** RentKit — онлайн‑каталог аренды квартир, домов и апартаментов.  
**Тип:** продуктовый лендинг + функциональный каталог + карточка объекта.  
**Цель:** показать навыки сложной адаптивной сетки, карточек с большим количеством данных, фильтров, галерей изображений, skeleton loading и интерактивных UI‑компонентов.
**Дизайн:** должен быть не темным и не слишком светлым. Должен соответствовать ощущению ярким, красивым видам из окон арендуемых объектов.


**Стек:**
- HTML5 (семантика + доступность)
- CSS (ITCSS + BEM)
- Vanilla JavaScript (фильтры, сортировка, галерея)
- Без препроцессоров, без сборщиков CSS, только нативные технологии.
- `<picture>` + WebP/AVIF
- GitHub Pages / Vercel

RentKit должен полностью имитировать настоящий сервис аренды.

## **3. Основной функционал**
### **3.1. Каталог недвижимости**
Каждый объект содержит:
- Изображение (основное)
- Название / тип: квартира / дом / студия
- Цена в месяц
- Адрес
- Количество комнат
- Площадь
- Статус: доступно / занято

### **3.2. Карточки каталога**
Содержат:
- `<picture>` для оптимизации изображений
- Фиксированное соотношение сторон через `aspect-ratio`
- Бейджи (например: *New*, *Premium*)
- Hover‑состояния: подъем карточки, тень
- Кнопка «Подробнее» → ведёт на `property.html`

### **3.3. Фильтры (JS)**
Фильтры работают без перезагрузки страницы:
- Цена: min/max (input range + number)
- Количество комнат (селект)
- Тип недвижимости (чекбоксы)
- Доступность
- Кнопка сброса фильтрации

Реакция должна быть мгновенной.

### **3.4. Сортировка**
- По цене (возрастание / убывание)
- По дате добавления

### **3.5. Skeleton loading**
При первой загрузке каталога:
- показываются 6–9 скелетон‑карточек
- shimmer‑анимация
- замена на реальные данные через 1–1.2 сек

### **3.6. Пагинация (опционально)**
- Простая кнопка «Показать ещё»
- Заглушка: догружаются новые объекты (mock)

## **4. Страница объекта (property.html)**
### **4.1. Галерея изображений**
- `<picture>` + WebP
- Слайдер - кастомный JS
- Thumbnails

### **4.2. Основная информация**
- Заголовок + адрес
- Цена
- Таблица характеристик: площадь, этаж, комнаты, год постройки
- Бейджи (новое, премиум)

### **4.3. Описание**
- Структурированный текст
- Адаптивная типографика

### **4.4. Карта**
Можно сделать:
- статичное изображение карты
- или iframe (не обязательно)

### **4.5. Форма заявки на просмотр**
Поля:
- Имя
- Телефон
- Комментарий
- Согласие на обработку данных

Отправка:
- fetch → mock endpoint
- состояния: loading / success / error
- aria-live уведомления

---

## **5. Адаптивность**
Подход: mobile-first.

### Breakpoints:
- <600px — мобильные устройства
- 600–1024px — планшеты
- >1024px — десктоп

### Изменения:
- Каталог: 1 → 2 → 3–4 колонки
- Хедер: бургер‑меню на мобильных
- Страница объекта: галерея над контентом на мобиле, сбоку — на десктопе

---

## **6. Анимации и UX**
- Scroll reveal ключевых секций
- Hover‑эффекты только через transform/opacity
- Плавные transition 150–200ms
- Prefers-reduced-motion: отключение сложных анимаций
- Плавное раскрытие FAQ (аккордеон)

---

## **7. Доступность (a11y)**
- Чёткая семантика: header, main, nav, section, footer
- aria-expanded для фильтров/аккордеонов
- Управление карточками с клавиатуры
- Контрастные состояния focus
- Галерея доступна для клавиатуры
- Поля формы с label

---

## **8. Оптимизация и перформанс**
- WebP/AVIF изображения
- Lazy-loading всех изображений каталога
- aspect-ratio для всех карточек (устранение CLS)
- Минификация CSS/JS
- Данные каталога могут храниться в отдельных JSON‑файлах (mock)
- Цель Lighthouse: Accessibility 100 / Performance 90+

---

## **9. Критерии готовности**
1. Каталог полностью адаптивен.
2. Фильтрация и сортировка работают.
3. Карточки корректно отображают все данные.
4. Скролл‑анимации не мешают UX.
5. Skeleton loading реализован.
6. Страница объекта работает полностью.
7. Форма отправляется через fetch.
8. Доступность: все элементы доступны с клавиатуры.
9. Performance оптимизирован (lazy-loading, aspect-ratio).
10. Деплой доступен по публичному URL.

---

## **11. Итог**
RentKit должен выглядеть как настоящая система аренды: мощный каталог, реальная карточка объекта, фильтры, галерея и форма заявки.  
Проект демонстрирует профессиональный уровень верстки, адаптива, логики интерфейсов и компонентного подхода.
